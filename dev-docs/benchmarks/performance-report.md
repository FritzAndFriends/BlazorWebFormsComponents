## Methodology

1. Each application is launched fresh before measurement.
2. **Cold start**: The first HTTP request after launch (includes JIT, initialization).
3. **Warm average**: Mean response time over N sequential requests (default 50).
4. **P95**: 95th percentile response time from warm requests.
5. .NET Framework apps run under IIS Express; Blazor apps use `dotnet run`.
6. All requests use `Invoke-WebRequest -UseBasicParsing` (no JS execution).

> **Note:** Results are machine-dependent. Focus on _relative_ comparisons,
> not absolute numbers.

- Results vary by machine — focus on relative differences.
- No concurrent-load testing (single sequential client).

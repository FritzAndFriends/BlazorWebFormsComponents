### Failed Test Details

**`StudentsPage_AddNewStudentFormWorks`**
- **Error:** `After adding student 'TestFirst2781 TestLast3244', expected the GridView to show the new entry. Rows before: 4, after: 4`
- **Root Cause:** `GetJoinedTableData()` uses `SelectMany` on enrollments, meaning students without enrollments are excluded from the grid. The `InsertNewEntry` BLL method creates the student and enrollment in a single transaction, but the test's Playwright fill/click timing may not trigger Blazor's `@bind-Text` change events before the button click fires. This is a Blazor binding timing issue with the BWFC `<TextBox>` component + Playwright `FillAsync`.
- **Impact:** Low — add form CRUD is the most complex interactive scenario and the underlying data layer logic is correct.
- **Fix:** Consider adding `@bind:event="oninput"` to TextBox bindings, or add a brief delay between fill and click in the test.

## What Didn't Work Well / Needs Attention

### 1. BWFC TextBox + Playwright Binding Timing
Playwright's `FillAsync` may not reliably trigger Blazor's `@bind` change event before a subsequent button click. This affects CRUD forms where multiple bound fields need to update before form submission. Consider:
- Adding `@bind:event="oninput"` support guidance in migration skills
- Documenting Playwright timing patterns for BWFC interactive tests

### 2. IDE0007 Style Rules Treated as Errors
The repository's `.editorconfig` enforces `var` usage as errors (IDE0007). The L2 transforms initially generated explicit type declarations, requiring an additional build cycle. Migration skills should note: "Use `var` for all local variable declarations."

### 3. ContentTemplate Warning
Three pages generate `RZ10012` warnings for `<ContentTemplate>` — this is a remnant from Web Forms `UpdatePanel` that the L1 script preserved. The L1 script or L2 transforms should strip `<ContentTemplate>` wrapper elements.

### 4. Students Page GetJoinedTableData Query
The BLL `GetJoinedTableData()` method uses `SelectMany` which excludes students without enrollments. This is a faithful translation of the original Web Forms logic, but it means newly added students don't appear in the grid until they have at least one enrollment.

## Recommendations for Next Run

1. **Strip `<ContentTemplate>` in L1 script** — Add a transform to remove `<ContentTemplate>` and `</ContentTemplate>` wrapper elements (remnants of `UpdatePanel`). This eliminates the RZ10012 warnings.

2. **Add `@page "/"` to Home page in L2 transforms** — The L1 script generates `@page "/Home"` from `Home.aspx` but doesn't add the root route. L2 should always add `@page "/"` to the first page.

3. **Add `<PageTitle>` to all pages in L2 transforms** — Web Forms pages inherit title from `<asp:Content ContentPlaceHolderID="TitleContent">`. L2 should generate `<PageTitle>` from this.

4. **Document Playwright timing for BWFC interactive tests** — The Students add form failure is a test timing issue. Document recommended patterns: add `await Task.Delay()` between fills and clicks for complex forms.

5. **Add `var` usage note to migration-standards skill** — Repos with strict `.editorconfig` rules (IDE0007) require `var` for all local declarations. This should be a standard L2 transform note.

6. **Handle element `id` preservation in L2** — Web Forms control IDs (like `dropListCourses`) should be preserved as `id` attributes on the replacement elements to maintain test compatibility.

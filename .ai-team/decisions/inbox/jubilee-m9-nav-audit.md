# WI-12: Sample Site Navigation Audit — Findings

**Audited by:** Jubilee (Sample Writer)
**Date:** 2026-02-25
**Scope:** Cross-reference of sample `.razor` pages on disk vs. `ComponentCatalog.cs` sidebar navigation entries

## Summary

The sidebar navigation is driven entirely by `ComponentCatalog.cs` (consumed by `NavMenu.razor`). Many sample pages exist on disk with valid `@page` directives but are **not reachable from the sidebar** because they are missing from `ComponentCatalog.cs` — either as main entries or as SubPage entries.

Some unreachable pages are partially discoverable via in-page `Nav.razor` components (GridView, FormView, ListView, DataList, DataGrid), but users must already be on a sibling page to see those links.

## Section 4: Additional Notes

1. **DataList "Flow" SubPage mismatch:** Catalog lists `"Flow"` as a SubPage, but the file on disk is `SimpleFlow.razor` with route `/ControlSamples/DataList/SimpleFlow`. The nav link would resolve to `/ControlSamples/DataList/Flow` which likely 404s.

2. **LoginControls `OrientationSample.razor`** exists at `/ControlSamples/LoginControls/Orientation` but is not linked from any nav. `LoginStatusNotAuthenticated.razor` also exists but the catalog only links to `LoginStatusAuthenticated`.

3. **MasterPage route casing:** Catalog uses `/control-samples/masterpage` (lowercase) while all other routes use `/ControlSamples/...` (PascalCase). This works due to case-insensitive routing but is inconsistent.

---

## Recommended Fix

Update `ComponentCatalog.cs` to add:
- 4 missing component entries (Menu, DataBinder, PasswordRecovery, ViewState)
- 15 missing SubPage entries across GridView, TreeView, FormView, DetailsView, ListView, DataGrid, Panel
- Fix DataList "Flow" → "SimpleFlow" SubPage name

This is a single-file change to `samples/AfterBlazorServerSide/ComponentCatalog.cs`.

### 2026-03-11: SelectMethod and database provider enforcement in skill files
**By:** Beast
**What:** Hardened all three migration skill files (`migration-standards/SKILL.md`, `bwfc-data-migration/SKILL.md`, `bwfc-migration/SKILL.md`) to prevent two recurring agent mistakes: (1) defaulting to SQLite instead of preserving the original SQL Server LocalDB provider, and (2) converting `SelectMethod` attributes to `Items=` binding instead of preserving them as `SelectHandler<ItemType>` delegates. Removed "Prefer SQLite for local dev" guidance, added "NEVER default to SQLite" warnings in two files, removed the "Alternatively bypass SelectMethod" escape hatch, added MANDATORY/MUST admonitions for SelectMethod preservation in all three files, and restricted `Items=` binding to `DataSource`-originating patterns only.
**Why:** ContosoUniversity migration was repeatedly contaminated with SQLite despite explicit instructions to use SQL Server LocalDB. About.razor and Students.razor had SelectMethod converted to Items= binding despite BWFC supporting SelectMethod natively. Root cause in both cases was permissive skill file wording — "Prefer SQLite" and "Alternatively… set Items directly" gave agents an escape path. Skill files are the primary instruction source for migration agents; ambiguous or permissive phrasing causes systematic regression across every run.

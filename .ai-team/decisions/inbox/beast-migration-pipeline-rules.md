### 2026-03-11: Migration pipeline enforcement in skill docs
**By:** Beast
**What:** Updated `migration-toolkit/skills/bwfc-migration/SKILL.md` and `migration-toolkit/skills/migration-standards/SKILL.md` to enforce the mandatory L1→L2 migration pipeline. Added prominent "Migration Pipeline — MANDATORY" section with critical warning admonition, exact `bwfc-migrate.ps1` invocation command, Layer 2 Copilot transform checklist, and explicit pipeline rules. Fixed all `TItem` references to `ItemType` throughout both skill files. Renamed "Layer 2 (Manual)" to "Layer 2 (Copilot-Assisted)" in migration-standards.
**Why:** Jeff Fritz directive: "When Copilot runs the migration skill, BOTH Layer 1 AND Layer 2 MUST be run. Copilot is NOT to do ANY code fixes between Layer 1 and Layer 2." This protects pipeline quality measurement — manual fixes between layers corrupt the signal on whether the L1 script needs improvement. The `ItemType` standardization aligns with the team-wide decision to match Web Forms `DataBoundControl.ItemType` naming.

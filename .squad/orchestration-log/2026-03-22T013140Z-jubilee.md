## What Happened

Fixed critical bug in `Employees.aspx.cs` search functionality. The page was using `FindControl()` to locate TextBox controls at runtime, which fails in Blazor. Replaced with protected backing fields initialized during component setup.

## Changes Made

- **File:** `samples/ContosoUniversity/Pages/Employees.aspx.cs`
- **Root Cause:** `FindControl()` depends on Web Forms control tree; not available in Blazor
- **Solution:** Added `SearchBox` and `NameStartsWith` as protected fields, populated in `OnPreRender()` before binding logic runs
- **Testing:** Build passed; verified no runtime failures

## Decision Made

**When binding data to components in Blazor:**
- Use protected component properties to hold control references
- Initialize in `OnPreRender()` hook, not `Page_Load()`
- This pattern replaces Web Forms' post-instantiation `FindControl()` calls

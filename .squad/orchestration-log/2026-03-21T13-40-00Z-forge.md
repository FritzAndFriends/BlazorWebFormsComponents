## Task

Scope a comprehensive milestone plan for **DepartmentPortal**, a new .NET Framework 4.8 sample application showcasing ASCX user controls, custom base classes, and enterprise Web Forms patterns to test migration toolkit coverage.

## Work Completed

### 1. Analysis Phase
- Identified current gaps in existing samples (BeforeWebForms, WingtipToys, ContosoUniversity)
- All existing samples have minimal ASCX usage; zero custom base classes
- Uncovered untested migration patterns:
  - ASCX → Blazor component migration
  - Custom Page/MasterPage/UserControl base classes
  - Nested user controls
  - ITemplate patterns
  - Web.config tagPrefix registration
  - Custom event args
  - ViewState/Session usage in controls

### 2. Application Design
- **Domain:** DepartmentPortal (internal HR/IT portal)
- **Justification:** Component-rich, enterprise-relevant, justifies custom base classes
- **Architecture:**
  - 12 ASCX user controls (simple display, data-bound, input with events, complex)
  - 3 custom base classes (BasePage, BaseMasterPage, BaseUserControl)
  - 14 pages (2 public, 9 authenticated, 3 admin, 1 master)
  - EF6 EDMX with 6 entities
  - Bootstrap 3 UI (matches era of previous samples)
  - Session-based auth (no ASP.NET Identity)

### 3. Control Catalog
Designed 12 ASCX controls covering full Web Forms feature spectrum:
- **Simple display:** Breadcrumb, PageHeader, Footer, QuickStats
- **Data-bound:** AnnouncementCard, EmployeeList, TrainingCatalog
- **Input with events:** SearchBox, DepartmentFilter, Pager
- **Complex patterns:** DashboardWidget (ITemplate), ResourceBrowser (nested)

### 4. Work Breakdown (4 Phases)
- **Phase 1:** Foundation (Jubilee) — .NET 4.8 project, EF6 EDMX, base classes, Site.Master
- **Phase 2:** ASCX Controls (Jubilee) — all 12 controls with code-behind, tagPrefix registration
- **Phase 3:** Pages (Jubilee) — all 14 pages, authentication, event handlers, ViewState/Session
- **Phase 4:** Testing & Documentation
  - 4.1 Manual smoke test (Jubilee)
  - 4.2 Migration coverage analysis (Bishop)
  - 4.3 Documentation (Beast)
  - 4.4–4.5 Acceptance/Unit tests (DEFERRED)

### 5. Risk Analysis
Identified 3 high-impact risks:
- **R1:** Toolkit may not support ASCX → Blazor (document patterns)
- **R2:** Custom base classes have no Blazor equivalent (design patterns)
- **R3:** ITemplate not supported in Blazor (RenderFragment approach)

### 6. Success Criteria & Timeline
- MVP targets: All 12 ASCX render, base classes functional, events work, ViewState/Session persist, nested/ITemplate/tagPrefix work
- Migration coverage: Toolkit executed, gaps documented, backlog created
- **Total timeline:** 7–11 days (1.5–2 weeks)
